# Style Profile

## Stable traits from current non-draft posts

- 中文为主，保留必要英文术语，不刻意全中文化。
- 风格偏工程记录，不写宣传腔，不写过度总结，不用空泛抒情句。
- 目标明确：记录一个真实问题、给出可执行步骤、顺手解释原因。
- 经常结合真实环境：Windows / Ubuntu / Android / Jetson / Atlas / Codex / Claude Code / MNN / HuggingFace。
- 常把命令、路径、配置项、类名、函数名、模型名写成行内代码。
- 喜欢把内容拆成分节结构，如 `## 1.`、`### 1.1`，而不是长段落连续叙述。
- 工具类常按 `安装 -> 配置 -> 使用 -> 常见命令/示例` 展开。
- 教程类常按 `背景 -> 前置知识 -> 环境准备 -> 分步骤操作 -> 验证` 展开。
- 系列首页常写总览、阶段划分、状态表、相关文章入口。
- 论文/源码类常按 `问题背景 -> 核心机制 -> 关键结构/代码路径 -> 自己的理解` 组织。
- 经常用表格总结“文章/说明/状态”、“阶段/内容/文章数”、“命令/说明”。
- 命令块可直接复制执行，尽量避免只给抽象描述。
- 会补充平台差异、版本细节、容易踩坑的点。

## Typical article categories

- 工具配置 / 命令备忘
- 环境搭建 / 开发流程记录
- 教程系列首页
- 论文阅读笔记
- 源码解析
- 刷题总结 / 知识点归纳

## Tone constraints

- 直接进入问题，不铺垫过多背景故事。
- 解释够用即可，不强行追求“完整教材化”。
- 遇到未完成内容，用 `TODO` / `WIP` / `草稿` 明确标识，不伪装成完整文章。
- 用“本文记录…/这里整理…/下面是…”这类自然开头，不用营销口吻。
